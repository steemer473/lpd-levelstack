## What this page is

- **Route:** `/reports/[reportId]` (dynamic; no `/reports` index)
- **Product:** LevelStack — paid diagnostic ($497) for small business owners
- **Job:** Scannable audit: grade, critical issue, section findings with plain-language guides, prioritized actions
- **Audience:** Non-technical owner; reads once, acts on "This week"
- **Tone:** Premium consultant report — not a SaaS dashboard

## User journey

1. Intake complete → redirect to `/reports/{id}`
2. **Generating** — 6 pipeline steps, ~2–3 min, status polling
3. **Ready** — `LevelstackReportView` from `report_json`
4. **Errors** — not found / failed / invalid data (centered cards)

**Shell:** Light gradient page, glass nav + sign out, `max-w-6xl` main, site footer. Report card owns the dark header — no second full-page hero.

## Tab model (7 tabs — important)

Default tab: **`executive_summary`**

| id | label | Notes |
|----|--------|--------|
| `executive_summary` | Executive summary | Summary + dashboard **on same tab**; tab dot = orange |
| `search_footprint` | Search footprint | AI preview + upsell strip |
| `online_reputation` | Reputation | |
| `digital_presence` | Digital presence | Often has scoreRows |
| `revenue_funnel` | Revenue funnel | Badge: "Funnel readiness: N%" |
| `competitive_context` | Competitive context | competitiveGrid table |
| `action_plan` | Action plan | Not scored; tab dot = amber |

Tab UI (**white bar below header**, per screenshot): icons + labels + (i) each; active tab = dark text + **orange** underline and orange icon. Inactive = gray. **Not** on the dark navy band. Selecting tab scrolls report card to top.

## UI blocks (ready state)

### 1. Dark header band (see `levelstack-executive-summary-v2.png`)
- Row 1: `{ownerName} · {businessName}` + subtitle `DIGITAL PRESENCE & REVENUE FUNNEL ASSESSMENT` (cyan, uppercase) + (i)
- Row 1 right: score box (`{overallScore}/100`, readiness label) + **square** orange-bordered grade box (`{letterGrade}`)
- Row 2 left: **3 stacked meta columns** (uppercase micro-label + bold value) with **vertical dividers** — Market, Report Date, Assessment Type
- Row 2 right: stat pills (findings, critical, sections)
- Background: `--lpd-dark` (`#002147`) — flat navy, no hero mesh
- Tab bar below on **white** (7 tabs) — not on the dark band

### 2. How to read (collapsible)
- Title: **"How to read this report"**
- 2 body paragraphs + italic diagnostic disclaimer
- Accordion directly under header (`aria-expanded`)

### 3. Executive summary tab (see screenshot — `executive-summary-v2.tsx`)
- **AI EXECUTIVE ASSESSMENT**: 3 insight columns (What Prospects See, Reputation Gap, Revenue Risk) + 3 cards (Critical Issue / Business Impact / Opportunity)
- **WHAT TO DO FIRST**: 0–30 / 31–60 / 61–90 day columns with checklists and colored focus pills
- Bottom grid: Competitive Snapshot + Key Strengths & Opportunities
- Do **not** use the old v1 pattern (paragraphs + readiness dashboard only)

### 4. Section tabs
- Header: title + **(i) guide** + status badge
- **Finding cards** (`DataPanel` pattern):
  - Micro-label (uppercase)
  - Value headline — key phrase in **orange** (`FindingValueHeadline`)
  - Parsed detail: SERP results (#position orange, links), key-value, bullets, paragraphs
  - Severity flag pill (Critical/High/Attention/Good + icon)
- Optional: **Website & presence signals** (score rows — value + bar + %)
- Optional: **AI search visibility preview** (2–3 col DataPanel grid)
- Optional: **Competitive table** (you column red-tinted)
- Search footprint: **SEO Automator Pro** upsell strip

### 5. Action plan tab
- Header + item count badge
- Groups: This week / This month / This quarter
- Table: # | Action (task, sub, findingRef) | Who | Time
- Closing upsell strip

### 6. Footer navigation
- **Previous** / **Next** section buttons (disabled at ends)
- `{n} of 7` counter

### 7. Report footer
- Generated by **LevelStack** · Level Play Digital
- As of `{reportDate}`

### 8. Scroll to top
- Fixed FAB bottom-right after ~600px scroll

## Section guide popovers (i)

Every panel title can show an Info button opening:
- **What this section shows** — plain language (often rich: bullets, callouts, orange accent terms)
- **Why it matters for your business**

Guides exist for: `executive_summary`, all 5 sections, `action_plan`.

## Data model (do not invent fields)

```ts
meta: {
  businessName, ownerName, marketLabel, reportDate, planId,
  overallScore, letterGrade,
  totalFindings, criticalCount, highCount, mediumCount, lowCount
}
executiveSummary: { paragraphs[], criticalIssue, firstSteps[] }
sections[]: {
  id, label, status: "critical"|"attention"|"good", score: 0-100,
  findings: { label, value, detail, severity }[],
  aiPreview?: { platform, result, severity }[],
  scoreRows?: { label, value, percent, tone: "red"|"amber"|"green" }[],
  competitiveGrid?: { columnHeaders[], rows: { label, cells[], youColumnIndex? }[] }
}
actionPlan: { thisWeek[], thisMonth[], thisQuarter[] }
// action: { task, sub?, who, time, findingRef? }
```

Finding `detail` strings may encode SERP JSON, bullet lists, or `key: value` lines — UI parses and renders structurally.
